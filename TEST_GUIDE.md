# 🧪 测试指南

## ✅ 服务状态

**服务地址：** http://localhost:8000

**启动命令：** 
```bash
java -Xmx2G -jar target\jpeg2pdf-ofd-1.0.0.jar
```

---

## 📋 测试项目

### 1️⃣ **Watch Folder 自动监控测试**

**步骤：**
```
1. 打开浏览器：http://localhost:8000
2. 找到「📂 Watch Folder 自動監控」區塊
3. 點擊右側開關（OFF → ON）
4. 觀察開關變成綠色，顯示 "ON"
5. 狀態文字變為「✅ 監控運行中」
```

**測試監控功能：**
```
1. 開啟監控後
2. 複製一張圖片到：C:\OCR\Watch
3. 等待 1-2 秒
4. 檢查輸出：C:\OCR\Output
   - 應該有 3 個文件：
     * {檔名}_{時間}.ofd
     * {檔名}_{時間}.pdf
     * {檔名}_{時間}.txt
5. 檢查原圖：C:\OCR\Save
   - 圖片應該被移動到這裡
```

**預期結果：**
```
✅ 開關可以正常切換
✅ ON/OFF 狀態清晰顯示
✅ 圖片自動處理
✅ 3 種格式同時導出
✅ 原圖自動移動
```

---

### 2️⃣ **OCR 進度表測試**

**步驟：**
```
1. 在主頁面上傳區域
2. 拖放一張圖片
3. 點擊「上傳並開始 OCR」
4. 觀察進度條出現
```

**預期結果：**
```
✅ 進度條顯示
✅ 進度百分比更新
✅ 狀態文字顯示（上傳中 → OCR中 → 完成）
✅ 詳細信息顯示
```

**進度階段：**
```
1. 📤 上傳中... (0-20%)
2. 🔍 OCR 識別中... (20-60%)
3. 📄 導出文件中... (60-90%)
4. ✅ 完成！ (100%)
```

---

### 3️⃣ **默認輸出格式測試**

**檢查：**
```
1. 查看輸出格式下拉選單
2. 第一個選項應該是「Searchable PDF」
3. 應該有 selected 屬性
```

**格式列表：**
```
✅ Searchable PDF（默認）
✅ Searchable OFD
✅ 純文字 (TXT)
```

---

### 4️⃣ **開關按鈕尺寸測試**

**檢查：**
```
1. 觀察 Watch Folder 開關按鈕
2. 按鈕尺寸應該是：10×20（適中）
3. 圓形滑塊尺寸：8×8
4. 不應該太大或太小
```

**尺寸對比：**
```
修改前：14×28（太大）
修改後：10×20（適中）✅
```

## 🔍 故障排除

### 問題 1：Watch Folder 不工作

**檢查：**
```
1. 確認監控已開啟（開關顯示 ON）
2. 檢查 C:\OCR\Watch 資料夾是否存在
3. 查看後端日誌是否有錯誤
```

**解決方法：**
```
- 重新啟動服務
- 確認資料夾權限
- 查看日誌：logs/application.log
```

---

### 問題 2：進度條不顯示

**檢查：**
```
1. 確認已選擇文件
2. 確認已點擊上傳按鈕
3. 檢查瀏覽器控制台錯誤
```

---

### 問題 3：文件沒有導出

**檢查：**
```
1. 確認 OCR 成功執行
2. 檢查 C:\OCR\Output 資料夾權限
3. 查看後端日誌
```
