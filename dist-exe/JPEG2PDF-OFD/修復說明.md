## 📝 **配置選項**

### **輸入配置**

```json
{
  "input": {

    "pattern": "*.jpg",             // 文件模式
```

```json

}
```

**支持語言**：

- `ch` - 簡體中文
- `en` - 英文
- `japan` - 日文

## ⚙️ **技術細節**

### **文件結構**

```
dist-exe\JPEG2PDF-OFD\
├── JPEG2PDF-OFD.exe          (主程式，433 KB)
├── JPEG2PDF-OFD.ico          (圖標)
├── app\
│   ├── jpeg2pdf-ofd-1.0.0.jar  (應用 JAR，83.32 MB)
│   └── JPEG2PDF-OFD.cfg        (配置文件)
├── runtime\                    (JRE，~128 MB)
├── test-config.json           (測試配置)
└── 使用說明.bat               (使用說明)
```

### **主類配置**

```ini
[Application]
app.classpath=$APPDIR\jpeg2pdf-ofd-1.0.0.jar
app.mainclass=org.springframework.boot.loader.launch.JarLauncher  # Spring Boot 啟動器

[JavaOptions]
java-options=-Djpackage.app-version=1.0.0
java-options=-Xmx2G  # 2GB 內存
```

### **為什麼使用 JarLauncher？**

Spring Boot JAR 使用特殊的類加載器結構：
```
Main-Class: org.springframework.boot.loader.launch.JarLauncher
Start-Class: com.ocr.jpeg2pdf.CliApplication
```

`JarLauncher` 負責：
1. 設置類路徑
2. 加載 Spring Boot 依賴
3. 啟動實際的應用主類（`CliApplication`）

## 🔧 **故障排除**

### **問題 1：端口 8000 被占用**

**錯誤**：
```
Web server failed to start. Port 8000 was already in use.
```

**解決**：
```powershell
# 查找佔用端口的進程
Get-NetTCPConnection -LocalPort 8000

# 停止進程
Stop-Process -Id <PID> -Force
```

### **問題 2：找不到配置文件**

**錯誤**：
```
配置文件不存在
```

**解決**：
```cmd
# 確保配置文件存在
dir test-config.json

# 使用完整路徑
JPEG2PDF-OFD.exe D:\Projects\jpeg2pdf_ofd_cmd\dist-exe\JPEG2PDF-OFD\test-config.json
```

### **問題 3：OFD 生成失敗**

**錯誤**：
```
OFD文檔中沒有頁面
```

**狀態**：
- 這是已知問題
- 暫時使用 PDF 格式
- OFD 修復需要更多時間
